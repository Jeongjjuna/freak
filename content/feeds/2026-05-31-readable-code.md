AI 덕분에 코드 생산량은 훨씬 빨라졌다.

평소 확장성을 고려해 설계하는 것을 좋아했는데,
반복 구현이나 노가다에 가까운 테스트 코드 작성이 많이 줄어든 것 같아 좋다.

그래도 결국 버그가 발생하면 사람이 코드를 읽고 이해해야 하는 순간이 온다.
그렇다면 이왕이면 사람도 읽기 쉽게 설계하고 구현하는 것이 좋지 않을까?

AI에게 좋은 프롬프트를 주기 위해서도,
읽기 쉽고 확장 가능한 코드를 유지하기 위해서도,
개발자는 여전히 설계와 코드 품질을 고민해야 한다.

내가 이해하지 못하는 시스템을 AI에게 제대로 설명할 수는 없다.

어셈블리어가 아닌 고수준 언어를 사용하는 이상,
AI 시대에도 좋은 코드의 기준은 크게 달라지지 않은 것 같다.

오히려 더 중요해졌을지도.
