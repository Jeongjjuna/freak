> 참조 투명성(Referential Transparency) : 함수형 프로그래밍의 핵심 원칙

1. 순수성
- 규칙1. 값을 반환해야 한다.
    - 값 반환외에는 부수효과 없도록
        - 부수효과 : 파일 입출력, 데이터 전송,  화면 쓰기
    - 부수효과 없는 순수 애플리케이션이 가능한건가? 당연 불가능
        - 단, 타입 시스템을 사용해서 부수 효과가 발생하는 시점을 명시적으로 표시하고, 명확하게 드러내는 방향으로 가야한다.

- 규칙 2. 결과가 입력에만 의존해아함.
    - 결과값 계산을 위해 입력(파라미터)외에 다른것을 사용하지 않아야함.
        - ex) 계산을 위해 전역, 싱글톤, 파일, 네트워크 소켓등 사용 x


2. 불변성
- 불변 데이터 구조를 사용하자.
- 순수함수 + 불변데이터 ⇒ `디버깅하고 오류 피하기가 쉬워짐.`
    - 함수형 코드를 읽을 때 상태를 신경쓰지 않아도됨.
    - 결국 상태 변경을 신경쓰며 프로그래밍을 하게될때, 사람은 실수를 하게 된다.(예측 하기 어려운 버그 발생)

3. 전체성
- 모든 입력값에 대해 유한 시간내에 결과 반환하는 함수
    - 무한루프를 가지지 말것
    - 모든 입력에대한 결과를 반환해라(`예외 보다는 결과값을 반환할 것`)


<br>

객체 인스턴스 생성의 메모리 CPU 사용량?
- JVM 객체 생성 비용이 그렇게 높지 않음
- 불변 객체를 사용하면 JIT 컴파일러와 가비지 컬렉터가 효율적으로 동작할 확률이 높음.
- 즉, 약간의 객체 생성 비용보다, `예측 가능성`, `테스트 용이성`, `동시성 안정성` 등을 우선으로 봄

<br>

결국 함수형 프로그래밍은
"코드를 더 똑똑하게 작성하는 방법"이라기보다,

"예측 가능한 시스템을 만드는 방법", "디버깅을 쉽게하고, 상태에 따른 예측하기 어려운 버그 가능성을 줄이는 방법."

에 가까운 것 같다.
