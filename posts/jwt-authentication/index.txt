<pre><code>Header.Payload.Signature

eyJhbGciOiJIUzI1NiJ9          // Header: 알고리즘
.eyJzdWIiOiIxMjMiLCJleHAiOjE3MDAwMDAwfQ  // Payload: claims

<p><strong>중요</strong>: Payload는 Base64 인코딩이지 암호화가 아니다. 민감 정보를 넣으면 안 된다.</p>
<h2 id="2-access-token--refresh-token-전략">2. Access Token + Refresh Token 전략</h2>
<p>Access Token만 사용하면:</p>
<ul>
<li>만료 시간을 짧게 설정 → 자주 재로그인</li>
<li>만료 시간을 길게 설정 → 탈취 시 장기간 악용</li>
</ul>
<p>해결책: 짧은 Access Token + 긴 Refresh Token</p>
<pre><code>Access Token: 15분
Refresh Token: 7일 (DB에 저장)
</code></pre>
