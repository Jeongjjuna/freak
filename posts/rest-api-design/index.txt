20:T2e1d,<h2 id="1-url-설계">1. URL 설계</h2>
<p>리소스는 명사, 동사는 HTTP 메서드로 표현한다.</p>
<pre><code># 나쁜 예
GET  /getUsers
POST /createUser
POST /deleteUser/1

# 좋은 예
GET    /users
POST   /users
DELETE /users/{id}
</code></pre>
<p>계층 관계는 슬래시로 표현한다.</p>
<pre><code>GET  /users/{userId}/orders          # 특정 사용자의 주문 목록
GET  /users/{userId}/orders/{orderId} # 특정 주문
POST /users/{userId}/orders          # 주문 생성
</code></pre>

<pre><code>200 OK          → 조회, 수정 성공
201 Created     → 생성 성공 (Location 헤더 포함)
204 No Content  → 삭제 성공

400 Bad Request      → 요청 형식 오류, 유효성 검증 실패

409 Conflict         → 중복 (이미 존재하는 이메일 등)
