29:T1691,<h2 id="1-spring-security는-필터-기반이다">1. Spring Security는 필터 기반이다</h2>
<p>Spring Security는 서블릿 필터(Servlet Filter)를 기반으로 동작한다. 모든 요청은 서블릿 컨테이너(Tomcat 등)의 필터 체인을 먼저 거치게 되며, Spring Security는 이 필터 체인에 자신의 필터들을 끼워 넣어 보안 처리를 수행한다.</p>
<h2 id="2-delegatingfilterproxy와-filterchainproxy">2. DelegatingFilterProxy와 FilterChainProxy</h2>
<p>Spring Security가 서블릿 필터 체인에 어떻게 참여하는지 이해하는 것이 중요하다.</p>
<ol>
<li><strong>DelegatingFilterProxy</strong>: 서블릿 필터로 등록되지만 실제 로직은 없다. 스프링 컨텍스트에서 관리되는 <code>FilterChainProxy</code> 빈에게 보안 처리를 위임한다.</li>
<li><strong>FilterChainProxy</strong>: 실제 보안 필터들을 순서대로 실행한다. <code>SecurityFilterChain</code>을 통해 요청 URL 패턴에 맞는 필터 목록을 결정한다.</li>
</ol>
<figure data-rehype-pretty-code-figure=""><pre style="background-color:#282A36;color:#F8F8F2" tabindex="0" data-language="text" data-theme="dracula"><code data-language="text" data-theme="dracula" style="display: grid;"><span data-line=""><span>Servlet Container</span></span>
<span data-line=""><span>  └── FilterChain</span></span>
<span data-line=""><span>      ├── Filter 1</span></span>
<span data-line=""><span>      ├── Filter 2</span></span>
<span data-line=""><span>      ├── DelegatingFilterProxy (Spring Security의 진입점)</span></span>
<span data-line=""><span>      │      └── FilterChainProxy (Bean)</span></span>
<span data-line=""><span>      │             ├── SecurityContextPersistenceFilter</span></span>
<span data-line=""><span>      │             ├── UsernamePasswordAuthenticationFilter</span></span>
<span data-line=""><span>      │             ├── ExceptionTranslationFilter</span></span>
<span data-line=""><span>      │             └── FilterSecurityInterceptor</span></span>
<span data-line=""><span>      └── Filter 3</span></span></code></pre></figure>
<h2 id="3-주요-보안-필터의-역할">3. 주요 보안 필터의 역할</h2>
<p>Spring Security에는 10개 이상의 필터가 기본으로 포함되어 있다.</p>
<ul>
<li><strong>UsernamePasswordAuthenticationFilter</strong>: 폼 로그인을 처리한다. 사용자 이름과 비밀번호를 인증한다.</li>
<li><strong>ExceptionTranslationFilter</strong>: 인증 및 인가 예외(AuthenticationException, AccessDeniedException)를 처리한다.</li>
<li><strong>FilterSecurityInterceptor</strong>: 인가(Authorization)를 결정하는 마지막 필터다.</li>
</ul>
